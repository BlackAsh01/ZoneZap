# ZoneZap AI Engine - Training Guide

## 🎯 Training on Real Data

This guide shows you how to train the anomaly detection model using **actual location data** from your Firebase Firestore database.

---

## 📋 Prerequisites

1. **Python 3.9+** installed
2. **Dependencies installed**: `pip install -r requirements.txt`
3. **Firebase project** with location data in `movement_logs` collection
4. **Firebase Service Account Key** (for production) OR **Firebase Emulator** (for local testing)

---

## 🔑 Step 1: Get Firebase Credentials

### Option A: Production Firebase (Recommended)

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Select your project: **zonezap-a6953**
3. Click **⚙️ Project Settings** → **Service Accounts** tab
4. Click **Generate New Private Key**
5. Save the JSON file as `firebase-service-account.json` in the `ai-engine/` directory

### Option B: Local Emulator (For Testing)

If using Firebase emulators:
```bash
# Start emulators first
cd ../backend
firebase emulators:start

# In another terminal, set environment variable
export FIRESTORE_EMULATOR_HOST=localhost:8080
```

---

## 🚀 Step 2: Collect Location Data

Before training, ensure you have location data:

1. **Run the mobile app** and use location tracking
2. **Let it collect data** for at least a few hours/days
3. **Check Firestore** → `movement_logs` collection has records

**Minimum recommended:** 100+ location points per user

---

## 🏋️ Step 3: Train the Model

### Quick Start (Auto-detect credentials)

```bash
cd ai-engine
python train_with_firebase.py
```

### Advanced Options

```bash
# Train on all users' data (default)
python train.py --firebase-cred firebase-service-account.json

# Train on specific user
python train.py --firebase-cred firebase-service-account.json --user-id USER_ID_HERE

# Limit number of records
python train.py --firebase-cred firebase-service-account.json --limit 10000

# Adjust expected anomaly rate (1% to 10%)
python train.py --firebase-cred firebase-service-account.json --contamination 0.03
```

### Using CSV File (Alternative)

If you exported data to CSV:

```bash
python train.py --csv movement.csv
```

- ✅ **Speed range** makes sense (0-10 m/s for walking/running)

## 🔧 Troubleshooting

### "Firebase initialization failed"

**Solution:**
- Check service account key path is correct
- Verify Firebase project ID matches
- For emulator: Ensure `FIRESTORE_EMULATOR_HOST` is set

### "Only X records found (minimum recommended: 100)"

**Solution:**
- Collect more location data by using the mobile app
- Reduce `--limit` if you have less data
- Use `--contamination` to adjust for smaller datasets

### "No data available for training"

**Solution:**
- Check Firestore has `movement_logs` collection
- Verify user_id is correct (if using `--user-id`)
- Try without `--user-id` to use all users' data
- Use `--csv` option with exported data

### Model detects too many/few anomalies

**Solution:**
- Adjust `--contamination` rate (lower = fewer anomalies)
- Check if data quality is good (no GPS errors)
- Review `training_report.json` for data statistics

## 📈 Best Practices

1. **Collect diverse data:**
   - Normal walking patterns
   - Different times of day
   - Various locations
   - Both normal and unusual movements

2. **Regular retraining:**
   - Retrain weekly/monthly as more data accumulates
   - Update model as user behavior patterns change

3. **Monitor performance:**
   - Track false positives/negatives
   - Adjust contamination rate based on results
   - Review alerts generated by the model

4. **Data quality:**
   - Ensure GPS accuracy is good (accuracy < 20m)
   - Filter out obvious GPS errors
   - Use data from multiple users for better generalization
